# Value Proposition Canvas Guide

Use this guide to define unique value consistently for MVP planning.

## Customer Profile

### Customer Jobs
- Define what users are trying to get done.
- Cover at least one functional job and one emotional or social job.

### Pains
- Define what frustrates or blocks users today.
- Include effort, risk, uncertainty, or cost.

### Gains
- Define outcomes users want to achieve.
- Include speed, confidence, quality, or control improvements.

## Value Map

### Products and Services
- List the MVP components that directly support the target job.
- Avoid listing future capabilities outside MVP scope.

### Pain Relievers
- Map each major pain to a clear relieving mechanism.
- Prefer specific behavior over abstract claims.

### Gain Creators
- Map each key gain to a concrete product effect.
- Prefer measurable gains where possible.

## One-Line UVP Formula
Use this format:

For [target user], [product] helps [core job] by [unique mechanism], unlike [current alternative].

## Quality Checks
- Every major pain has at least one pain reliever.
- Every key gain has at least one gain creator.
- UVP is specific enough to guide prioritization.
- UVP can be tested in a short validation cycle.
