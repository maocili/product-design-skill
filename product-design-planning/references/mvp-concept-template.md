# MVP Concept Template

Use this template to define a documentation-first MVP concept before implementation.

## 1. Problem and Context
- Problem statement: What concrete user pain are we solving?
- Target user: Who has this pain with enough urgency?
- Trigger moment: When does this pain become actionable?
- Current alternatives: What do users do today?

## 2. Concept Definition
- Product concept in 2-3 sentences.
- Intended user outcome in measurable terms.
- Intended business outcome in measurable terms.

## 3. Scope Boundary
- Must: Minimum capabilities needed to validate unique value.
- Should: Useful additions if they do not delay validation.
- Won't: Explicitly excluded in MVP.

## 4. Key User Flow
Describe the single end-to-end flow that proves the value proposition:
1. Entry
2. Input
3. Core value moment
4. Confirmation
5. Repeat trigger

## 5. Assumptions
List assumptions separately from facts. Mark each as:
- High confidence
- Medium confidence
- Low confidence

## 6. Completion Check
- Can a stakeholder explain the product concept in one sentence?
- Is the Must scope small enough for a short validation cycle?
- Is there exactly one primary user flow for MVP?
- Are assumptions explicit and testable?
