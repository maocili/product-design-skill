# Validation Checklist

Use this checklist to design the first MVP validation cycle.

## 1. Smallest Useful Experiment
- Define the minimum test that can confirm or reject the UVP.
- Keep experiment duration short (for example, 1-2 weeks).
- Focus on one target segment and one primary user flow.

## 2. Success Signals
- Define measurable success criteria before running the test.
- Include at least one behavioral metric and one outcome metric.
- Keep thresholds concrete and numeric.

## 3. Failure Threshold
- Define explicit conditions that indicate MVP value is not validated.
- Avoid subjective or post-hoc interpretation.

## 4. Decision Gate
- Define what happens if success criteria are met.
- Define what happens if signals are mixed.
- Define what happens if failure threshold is hit.

## 5. Metrics Baseline
- Activation: first successful completion rate.
- Time-to-value: time from entry to first value moment.
- Repeat intent or early retention.

## 6. Documentation Quality Checks
- Are assumptions listed separately from facts?
- Is every metric tied to a decision?
- Can the team run this experiment without extra discovery work?
